# 📄 Markdown2PDF Quality Gate

## Overview

This project implements an automated testing framework and CI/CD quality gate for validating PDF artifacts generated from Markdown documents using Markdown2Pdf.Console.

The system ensures that generated PDF documents are not only successfully created, but also structurally and semantically correct.

This work is developed as part of a Bachelor thesis in Business Informatics.

---

## Research Objective

The objective of this project is to evaluate how effectively automated testing techniques can detect structural and content-based regressions in generated PDF artifacts.

Additionally, the project investigates the use of Large Language Models (LLMs) for automated test generation.

---

##  Key Features

* Automated Markdown-to-PDF conversion testing
* Rule-based validation of PDF content
* Regression detection (missing or corrupted elements)
* LLM-assisted test generation
* CI/CD integration via GitHub Actions
* Automated test reporting (HTML reports)

##  Validation Concept

The system validates PDF artifacts by extracting text and applying rule-based checks.

### Example Rules

* Headings must be present (e.g., *Introduction, Features*)
* Tables must contain expected rows (e.g., *Alice, Bob*)
* Lists must include all elements
* Code blocks must be preserved

---

## Regression Scenarios

Regression scenarios simulate realistic document errors:

* Missing headings
* Broken tables
* Missing list items
* Missing code blocks

> Important:
> PDF generation still succeeds.
> Errors occur at the **content level**, not at the conversion level.

---

## Experimental Design

The project evaluates two testing approaches:

## Running Tests

Run all tests:

```id="run-all"
pytest -v
```

Run a specific file:

```id="run-file"
pytest -v tests/test_auto_conversion.py
```

Run a specific test:

```id="run-one"
pytest -v tests/test_auto_conversion.py::test_heading
```

Run regression tests:

```id="run-regression"
pytest -v -k "missing or broken"
```

---

## Generate Test Report

Generate HTML report:

```id="generate-report"
pytest -v --html=results/test-reports/report.html --self-contained-html
```

Open automatically (Windows):

```id="open-report"
pytest -v --html=results/test-reports/report.html --self-contained-html ; start results/test-reports/report.html
```

## Key Insight

This project distinguishes between:

* **Conversion success** → PDF is generated
* **Content validity** → PDF is correct

A document may pass conversion but still fail validation.
