# Markdown2PDF Quality Gate

## Overview
This project implements an automated integration test suite and CI/CD quality gate for validating PDF artifacts generated by Markdown2Pdf.Console.

The project is developed as part of a Bachelor thesis in Business Informatics. Its purpose is to detect structural and content regressions in generated PDF documents and to support automated quality assurance in document generation pipelines.

## Research Goal
The main goal is to evaluate how effectively automated integration tests can detect regressions in PDF artifacts generated from Markdown documents.

## Scope
This repository focuses on:
- automated Markdown-to-PDF conversion testing
- structural validation of generated PDF artifacts
- content validation of generated PDF artifacts
- regression detection using baseline comparison
- CI/CD integration through GitHub Actions

This repository does not implement the Markdown2Pdf.Console converter itself. Instead, it tests and evaluates the outputs generated by that tool.

## Repository Structure
- `data/`  
  Contains pilot dataset files, regression scenarios, and baselines.
- `tests/`  
  Contains the automated integration tests and helper modules.
- `results/`  
  Contains generated PDFs and test reports.
- `.github/workflows/`  
  Contains the CI/CD workflow configuration.

## Current Phase
The project is currently in the implementation phase. The focus is on:
1. defining validation rules
2. expanding the pilot dataset
3. implementing automated integration tests
4. preparing controlled regression experiments
